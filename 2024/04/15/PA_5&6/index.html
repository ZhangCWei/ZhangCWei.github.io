Iterative Algorithm, Another View

Iterative Algorithm for May & Forward Analysis

View Iterative Algorithm in Another Way

  • Given a CFG (program) with k nodes, the iterative algorithm updates OUT[n]OUT[n]OUT[n] for every node n in each iteration.
  • Assume the domain of the values in data flow analysis is V (所有的赋值，即分析的值域), then we can define a k-tuple (OUT[n1],OUT[n2],...,OUT[nk])(OUT[n_1],OUT[n_2],...,OUT[n_k])(OUT[n1​],OUT[n2​],...,OUT[nk​]) as an element of set (V1×V2…×Vk)(V_1 × V_2 … × V_k)(V1​×V2​…×Vk​) denoted as VkV^kVk (存储每个节点每次迭代后的 OUTOUTOUT 值), to hold the values of the analysis after each iteration.
  • Each iteration can be considered as taking an action to map an element of VkV^kVk to a new element of VkV^kVk, through applying the transfer functions and control-flow handing, abstracted as a function F:Vk→VkF: V^k → V^kF:Vk→Vk.
  • Then the algorithm outputs a series of k-tuples iteratively until a k-tuple is the same as the last one in two consecutive iterations.
  • XXX is a fixed point of function FFF if X=F(X)X = F(X)X=F(X), so the iterative algorithm reaches a fixed point.

Question

  • The iterative algorithm (or the IN/OUT equation system) produces a solution to a data flow analysis
    • Is the algorithm guaranteed to terminate or reach the fixed point, or does it always have a solution?
    • If so, is there only one solution or only one fixed point? If more than one, is our solution the best one (most precise)?
    • When will the algorithm reach the fixed point, or when can we get the solution?

Partial Order

  • We define poset as a pair (P,⊑)(P, ⊑)(P,⊑) where ⊑⊑⊑ is a binary relation that defines a partial ordering(偏序关系) over PPP, and ⊑⊑⊑ has the following properties:
    • ∀x∈P,x⊑x∀x ∈ P, x ⊑ x∀x∈P,x⊑x (Reflexivity, 自反性)
    • ∀x,y∈P,x⊑y∧y⊑x⟹x=y∀x, y ∈ P, x ⊑ y ∧ y ⊑ x ⟹ x = y∀x,y∈P,x⊑y∧y⊑x⟹x=y (Antisymmetry, 反对称性)
    • ∀x,y,z∈P,x⊑y∧y⊑z⟹x⊑z∀x, y, z ∈ P, x ⊑ y ∧ y ⊑ z ⟹ x ⊑ z∀x,y,z∈P,x⊑y∧y⊑z⟹x⊑z (Transitivity, 传递性)
  • Partial means for a pair of set elements in P, they could be incomparable (集合中任取两个值，它们可以是无法比较的); in other words, not necessary that every pair of set elements must satisfy the ordering ⊑⊑⊑.
  • Examples
    • Is (S, ⊑) a poset where S is a set of integers and ⊑ represents ≤ (less than or equal to) ?
      • Yes, x ≤ x; x ≤ y and y ≤ x => x = y; x ≤ y and y ≤ z => x ≤ z
    • Is (S, ⊑) a poset where S is a set of integers and ⊑ represents < (less than) ?
      • No, x < x is incorrent
    • Is (S, ⊑) a poset where S is a set of English words and ⊑ represents the substring relation, i.e., s1 ⊑ s2 means s1 is a substring of s2?
      • Yes. 单词是单词的子字符串；两个单词互为子字符串，必相等；子字符串具有传递性质。
    • Is (S, ⊑) a poset where S is the power set of set {a,b,c} and ⊑ represents ⊆ (subset) ?
      • Yes.

Upper and Lower Bounds

  • Given a poset (P, ⊑) and its subset S that S ⊆ P, we say that

    • u∈Pu ∈ Pu∈P is an upper bound of S, if ∀x∈S,x⊑u∀x ∈ S, x ⊑ u∀x∈S,x⊑u.
    • l∈Pl ∈ Pl∈P is an lower bound of S, if ∀x∈S,l⊑x∀x ∈ S, l ⊑ x∀x∈S,l⊑x.

  • We define the least upper bound (lub or join) of S, written ⊔S⊔S⊔S, if for every upper bound of S, say u, ⊔S⊑u⊔S ⊑ u⊔S⊑u. (最小上界join)

  • Similarly, We define the greatest lower bound (glb, or meet) of S, written ⊓S⊓S⊓S, if for every lower bound of S, say l, l⊑⊓Sl ⊑ ⊓Sl⊑⊓S. (最大下界meet)

  • Usually, if S contains only two elements a and b (S = {a, b}), then ⊔S⊔S⊔S can be written a⊔ba ⊔ ba⊔b (the join of a and b), ⊓S⊓S⊓S can be written a⊓ba ⊓ ba⊓b (the meet of a and b)

Some Properties

  • Not every poset has lub or glb, But if a poset has lub or glb, it will be unique.
  • Proof.
    • assume g1g_1g1​ and g2g_2g2​ are both glbs of poset P,
    • according to the definition of glb, g1⊑(g2=⊓P)g_1⊑(g_2=⊓P)g1​⊑(g2​=⊓P) and g2⊑(g1=⊓P)g_2⊑(g_1=⊓P)g2​⊑(g1​=⊓P),
    • by the antisymmetry of partial order ⊑, we know g1=g2g_1 = g_2g1​=g2​

Lattice, Semilattice, Complete and Product Lattice

Lattice

  • Given a poset (P, ⊑), ∀a, b ∈ P, if a ⊔ b and a ⊓ b exist, then (P, ⊑) is called a lattice.
    • A poset is a lattice if every pair of its elements has a least upper bound and a greatest lower bound (对于一个偏序集中的任意两个元素，都存在最小上界和最大下界，则称为格)
  • Example
    • Is (S, ⊑) a lattice where S is a set of English words and ⊑ represents the substring relation, i.e., s1 ⊑ s2 means s1 is a substring of s2?
      • 这并不是一个格，例如，pin 和 sin 没有最小上界

Complete Lattice

  • Given a lattice (P, ⊑), for arbitrary subset S of P, if ⊔S and ⊓S exist, then (P, ⊑) is called a complete lattice.
    • All subsets of a lattice have a least upper bound and a greatest lower bound (对于一个偏序集中的任意子集，都存在最小上界和最大下界，则称为完全格)
  • Every complete lattice (P, ⊑) has a greatest element ㄒ=⊔Pㄒ = ⊔Pㄒ=⊔P called top and a least element ⊥=⊓P⊥ = ⊓P⊥=⊓P called bottom.
  • Every finite lattice (P is finite) is a complete lattice. (有穷的格一定是完全格)
  • Example

    • Is (S, ⊑) a complete lattice where S is a set of integers and ⊑ represents ≤ (less than or equal to) ?

      • For a subset S+S^+S+ including all positive integers, it has no ⊔S+(+∞)⊔S^+(+∞)⊔S+(+∞)

    • Is (S, ⊑) a complete lattice where S is the power set of set {a,b,c} and ⊑ represents ⊆ (subset) ?

      • Yes. Note: the definition of bounds implies that the bounds are not necessarily in the subsets, but they must be in the lattice. (边界不一定在子集内，但它们必须在格中)

  • A product lattice is a lattice. 乘积格也是格。

  • If a product lattice L is a product of complete lattices, then L is also complete. 如果乘积格由多个完全格组成，那么这个乘积格也是一个完全格。

Data Flow Analysis Framework via Lattice

  • A data flow analysis framework (D, L, F) consists of :
    • D : a direction of data flow: forwards or backwards.
    • L : a lattice including domain of the values V and a meet ⊓ or join ⊔ operator.
    • F : a family of transfer functions from V to V.
  • Data flow analysis can be seen as iteratively applying transfer functions and meet/join operations on the values of lattice.

Monotonicity and Fixed Point Theorem

  • Monotonicity
    • A function f: L → L (L is a lattice) is monotonic (单调的) if
      ∀ x, y ∈ L, x ⊑ y ⟹ f(x) ⊑ f(y)
  • Fixed-Point Theorem
    • Given a complete lattice (L, ⊑), if
      • f: L → L is monotonic and
      • L is finite
    • Then:
      • the least fixed point of f can be found by iterating f(⊥), f(f(⊥)), … , f^k^(⊥) until a fixed point is reached
      • the greatest fixed point of f can be found by iterating f(ㄒ), … , f^k^(ㄒ) until a fixed point is reached
    • 不动点定理：给定一个有穷的完全格，若其转换函数是单调的，那么它的 least 不动点与 great 不动点就是第一个到达的不动点。
  • Proof 1:
    1. By the definition of ⊥ and f: L → L, we have ⊥ ⊑ f(⊥)
    2. As f is monotonic, we have f(⊥) ⊑ f^2^(⊥)
    3. Similarly, we have ⊥⊑ f(⊥) ⊑ … ⊑ f^k^(⊥)
    4. As L is finite, for some k, we have f^Fix^ = f^k^(⊥) = f^k+1^(⊥).
      Thus, the fixed point exists.
  • Proof 2 :
    1. Assume we have another fixed point x where x = f(x).
    2. By the definition of ⊥, we have ⊥ ⊑ x.
    3. As f is monotonic, we have f(⊥) ⊑ f(x).
    4. Assume f^i^(⊥) ⊑ f^i^(x), as f is monotonic, we have f ^i+1^(⊥) ⊑ f ^i+1^(x).
    5. Thus f^i^(⊥) ⊑ f^i^(x) = x, then we have f^Fix^ = f^k^(⊥) ⊑ x.
    6. Thus the fixed point is the least.
  • Now what we fixed point have just seen is the property (fixed point theorem) for the function on a lattice. We cannot say our iterative algorithm also has that property unless we can relate the algorithm to the fixed point theorem.

May/Must Analysis, A Lattice View

  • may 和 must analyses 都是一个从 unsafe 往 safe 的过程

May analysis

  • may 分析的目的可以看作是报错，因此初始化在 bottom，此时分析结果为无错误，因此肯定是一个 unsafe 的结果；而在 top 处，分析结果是所有点都出错，这个肯定是 safe 的，但却是无用功。因此我们的目的是在二者之间找到一个平衡点。
  • 假设{a,c}是truth，那么以此为界可以划分为两块(之所以是区域，是因为这其实是一个product lattice)，交界点便是truth。
  • 因此最后可以得到下右图，假设有一系列不动点，那么越往上精度越低，对于 may 来说最接近 truth 的就是最小不动点。

Must Analysis

  • must 的目的可以看作是删除优化，因此初始化在 top，此时分析结果为所有语句都可删除，那肯定是unsafe的 (上方会有误报)；而在 bottom 处，分析结果是所有语句不删除 (下方是Complete的)，这个肯定是 safe 的，但完全没有优化，也是无用功。

MOP and Distributivity

Meet-Over-All-Paths Solution (MOP)

P=Entry→S1→S2→...→SiP = Entry→S_1→S_2→...→S_i P=Entry→S1​→S2​→...→Si​

  • Transfer function FPF_PFP​ for a path PPP (from EntryEntryEntry to SiS_iSi​) is a composition of transfer functions for all statements on that path: fS1,fS2,...,fSi−1f_{S1}, f_{S2},...,f_{Si-1}fS1​,fS2​,...,fSi−1​

MOP[Si]=⊔/⊓A path P from Entry to SiFp(OUT[Entry])MOP[S_i] = \underset{A\ path\ P\ from\ Entry\ to\ S_i}{⊔/⊓} F_p(OUT[Entry]) MOP[Si​]=A path P from Entry to Si​⊔/⊓​Fp​(OUT[Entry])

  • MOP computes the data-flow values at the end of each path and apply join / meet operator to these values to find their lub / glb (把所有路径的结果结合起来)
  • Some paths may be not executable: not fully precise. (在动态输入中，可能存在无论输入什么都不会执行的路径)
  • Unbounded, and not enumerable: impractical. (循环停止条件不明、不可枚举)

Ours (Lterative Alogrithm) vs. MOP

  • Proof.
    • By definition of lub ⊔, we have: x ⊑ x ⊔ y and y ⊑ x ⊔ y
    • As transfer function F is monotonic, we have F(x) ⊑ F(x ⊔ y) and F(y) ⊑ F(x ⊔ y)
    • That means F(x ⊔ y) is an upper bound of F(x) and F(y)
    • As F(x) ⊔ F(y) is the lub of F(x) and F(y), we have F(x) ⊔ F(y) ⊑ F(x ⊔ y)
    • So MOP ⊑ Ours
  • Ours is less precise than MOP
  • But when F is distributive (可分配性), i.e., F(x ⊔ y) = F(x) ⊔ F(y)
    • Thus MOP = Ours, Ours is as precise as MOP
    • Bit-vector or Gen/Kill problems (set union/intersection for join/meet) are distributive
    • But some analyses are not distributive, like Constant Propagation

Constant Propagation

  • GIven a variable x at program point p, determine whether x is guaranteed to hold a constant value at p.
    • The OUT of each node in CFG, includes a set of pairs (x, v) where x is a variable and v is the value held by x after that node.
  • 在程序点 p 处的变量 x，判断 x 是否一定指向常量值。
    • 类别：must分析，因为要考虑经过 p 点所有路径上，x 的值必须都一样，才算作一定指向常量。
