How to Implement Pointer Analysis

  • Essentially, pointer analysis is to propagate points-to information among pointers (variables & fields); 本质上，指针分析是将指向信息传播到由变量和字段构成的指针之间的过程
  • Pointer analysis as solving a system of inclusion constraints for pointers
  • Key to implementation: when pt(x)pt(x)pt(x) is changed, propagate the changed part to the related pointers of xxx;
    • 为了实现指针分析，我们可以通过一张图来链接相关联的指针
    • 当 xxx 的指针域 pt(x)pt(x)pt(x) 改变的时候，将改变的部分传递给 xxx 的后继

Pointer Analysis: Algorithms

  • S：输入程序中的语句集合 (控制流不敏感)
  • WL：WorkList，包含了一系列的<指针，指针集>
    • Worklist contains the points-to information to be processed WL⊆<Pointer,P(O)>WL⊆<\text{Pointer}, P(O)>WL⊆<Pointer,P(O)>
    • Each worklist entry <n,pts><n, pts><n,pts> is a pair of pointer 𝑛𝑛n and points-to set ptsptspts, which means that ptsptspts should be propagated to pt(n)pt(n)pt(n)
  • PFG：指针流图
  • 初始化部分
    • WL 和 PFG 均初始化为空
    • 检测所有语句中的 New 语句，将指针与对应的域绑定传入 WL 中
    • 对每个 x = y 的 Assign 语句进行一次 AddEdge(y, x)
  • AddEdge(s, t) 函数
    1. 判断从 s 流向 t 的边是否已经在 PFG 中，若存在则无需进行任何操作；
    2. 若不存在则将 s→ts→ts→t 加入 PFG 中；
    3. 判断源点 s 的指针域是否为空：若不为空，就将其与目标点构成对加入 WL 中，确保 s 指向的指针域一定全部传递给 t
  • 主体部分

    1. 首先进行差分：先从 WL 中按照某种规则取出一个对 <n,pts><n, pts><n,pts>，取 ptsptspts 和 pt(n)pt(n)pt(n) 的差集，即去除 ptsptspts 中所有存在于 nnn 指针域中的指向信息，得到 ΔΔΔ

      差分是为了进行去重，减少冗余操作，降低系统的开支
    2. 对 nnn 和 ΔΔΔ 做一个 Propagate 传播操作
      • 传播函数做两件事：
        • 将传入的指针域 ptsptspts 加入 nnn 的指针域
        • 将 ptsptspts 与 nnn 的所有后继构成对加入 WL 中
      • 结合之前的 ΔΔΔ，可以理解为，将 nnn 缺少的指向信息加入 nnn 的指针域中，然后将这变化部分传播给 nnn 的后继
      • pt(n)⋃=ptspt(n) ⋃= ptspt(n)⋃=pts 是整个算法里唯一改变指针集的地方
    3. 处理 Store 和 Load 语句
      • 如果 nnn 是一个变量，对所有在 ΔΔΔ 中的指向信息，在程序语句集中找到与这个变量有关的 n.fn.fn.f，进行对应的 AddEdge 操作

Pointer Analysis with Method Calls

CHA and Pointer Analysis

  • Inter-procedural pointer analysis requires call graph
    • CHA: resolve call targets based on declared type of aaa
    • Pointer analysis: resolve call targets based on pt(a)pt(a)pt(a)
  • Call graph construction
    • CHA: imprecise, introduce spurious call graph edges and points-to relations;
      • 基于 a 的**声明对象类型 **A 处理调用信息，会引入虚假的调用边和指向关系
    • Pointer analysis: more precise than CHA, both for call graph and points-to relations;
      • 基于 a 的指针域 pt(a) 来处理调用信息，比较精确，因为用指针分析构造调用图，所以同时考虑到调用图和指向关系
      • 也称作 on-the-fly call graph construction

Rule

  • 根据指针分析规则，调用语句左侧的 r 由右侧指针域而不是 x 的声明对象类型决定。不同于 CHA 关心如返回地址等控制流信息，指针分析关注数据流。指针分析处理调用的时候一般负责如下四件事：
    1. 确定目标方法
    2. 传递 receiver object
    3. 传递参数
    4. 传递返回值
  • 符号定义
    • Dispatch(oi,k)Dispatch(o_i, k)Dispatch(oi​,k) : resolves the virtual dispatch of kkk on 𝑜𝑖𝑜_𝑖oi​ to a target method (based on type of 𝑜𝑖𝑜_𝑖oi​) ;
      • 基于 oio_ioi​ 的类型 (即实例对象 xxx 的类型)，将在 oio_ioi​ 上的虚拟调用解析为目标方法。在规则中就是解析为目标方法后赋予给 mmm。
    • mthism_{this}mthis​ : this variable of 𝑚𝑚m ;

Interprocedural Pointer Analysis

  • Run together with call graph construction
  • Call graph forms a “reachable world”
    • Entry methods (e.g., the main method) are reachable from the beginning
    • The other reachable methods are gradually discovered during analysis
    • Only reachable methods and statements are analyzed
