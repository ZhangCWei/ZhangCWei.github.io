1 自然语言处理概论

1.1 自然语言处理的基本概念

  • 语言是个体之间由于沟通需要而制定的指令

  • 自然语言通常是指一种自然地随文化演化的人类语言，与之相对的是人工语言如世界语、编程语言等

    • 自然语言通常有两种形式： 书面语 (Text) 和口语 (Speech)

  • 自然语言的特点

    • 线性：自然语言呈现为一种线性的符号序列
    • 层次性：自然语言内部存在层次结构
    • 歧义性：同一个自然语言句子存在多种不同的理解
    • 演化性：自然语言随着时代不断演化

  • 自然语言处理的主要难点：在各个层面广泛存在各种各样的歧义性

    • 语音歧义：主要体现在口语中， 是由于语言中同音异义词、爆破音不完全等原因造成的；
    • 词语切分歧义：由字符组成词语时的歧义现象；
    • 词义歧义：词语具有相同形式但是不同意义；
    • 结构歧义：词组成词组或者句子时，由于其组成的词或词组间可能存在不同的语法或语义关系而出现的 (潜在) 歧义现象；
    • 指代歧义：指代词 (如我，你，他等) 和代词词组 (如“那件事”，“这一点”等) 所指的事件可能存在歧义；
    • 省略歧义：自然语言中由于省略所产生的歧义；
    • 语用歧义：由于上下文、说话人属性、场景等语用方面的原因造成的歧义。

  • 自然语言处理 (Natural Language Processing，NLP) 旨在探索实现人与计算机之间用自然语言进行有效交流的理论与方法。

    • 能够理解自然语言的意义 (自然语言理解，NLU)
    • 以自然语言文本来表达给定的意图、思想等 (自然语言生成，NLG)
  • 自然语言处理历史
    • 符号学派和随机学派
    • 基于逻辑的范式、基于规则的范式和随机范式
    • 基于机器学习和数据驱动
    • 深度神经网络+向量表示
    • 超大规模语言模型

1.2 自然语言处理的基本范式

  • 自然语言处理的发展经历了基于规则的方法、基于机器学习的方法以及基于深度学习的方法三种范式，三种范式基本对应了自然语言处理的不同发展阶段的重点
    • 三种范式各有利弊，需要根据任务的特点、计算量、可控制性以及可解释性等具体情况进行选择
  • 基于规则的方法
    • 通过词汇、形式文法等制定的规则引入语言学知识，从而完成相应的自然语言处理任务
  • 基于机器学习的方法
    • 将自然语言处理任务转化为某种分类任务，可以说是在试图模拟人类完成某个任务时的思维过程；
    • 优点：直观、可解释、不依赖大规模数据。利用规则所表达出来的语言知识具有一定的可读性，不同的人之间可以相互理解；
    • 缺点： 覆盖率差、大规模规则构建代价大、难度高等。
  • 基于深度学习的方法
    • 将特征学习和预测模型融合，通过优化算法使模型自动地学习出好的特征表示，并基于此进行结果预测
  • 基于大模型的方法
    • 将大量各类型自然语言处理任务，统一为生成式自然语言理解框架；
    • ChatGPT 所展现出来的通用任务理解能力和未知任务泛化能力，使得未来自然语言处理的研究范式可能进一步发生变化；
    • 在大规模语言模型构建阶段，通过大量的文本内容，训练模型长文本的建模能力，使得模型具有语言生成能力，并使得模型获得隐式的世界知识；
    • 在通用能力注入阶段，利用包括阅读理解、情感分析、信息抽取等现有任务的标注数据，结合人工设计的指令词对模型进行多任务训练，从而使得模型具有很好的任务泛化能力；
    • 特定任务使用阶段则变得非常简单，由于模型具备了通用任务能力，只需要根据任务需求设计任务指令，将任务中所需处理的文本内容与指令结合，然后就可以利用大模型得到所需结果。

1.3 自然语言处理的常见任务

  • 分类问题
  • 匹配问题
  • 翻译问题
  • 结构预测问题
  • 马尔可夫决策问题

2 语言模型

2.1 ChatGPT介绍

  • ChatGPT 是 OpenAI 在2022年11月30日发布的聊天机器人模型，它能够通过学习和理解人类的语言来进行对话，还能根据聊天的上下文进行互动，甚至能完成撰写邮件、视频脚本、文案、翻译、代码等任务
    • Generative：生成有结构的复杂对象
    • Pre-trained：基于互联网数据的预训练
    • Transformer：注意力机制

  • ChatGPT 的由四个阶段组成

    1. 学习文字接龙

    2. 人类老师引导

    3. 模仿人类老师喜好

    4. 向模拟老师学习

2.2 语言模型基础

  • 从统计角度看，自然语言中的一个句子 s 可以由任何词串构成。不过 P(s) 有大有小，如：
    • a = 我准备去散步；b = 我去散步准备；那么，P(a) > P(b)
  • 对于给定的句子 s 而言，通常 P(s) 是未知的。对于一个句子空间 A，其概率分布 D 表示任意可能句子的概率分布。估计句子空间 A 的概率分布 D 的过程被称作语言建模。
  • 语言模型：目标是建模自然语言的概率分布，在自然语言处理研究中具有重要的作用，是机器翻译、语音识别、输入法、句法分析等任务的支撑。语言模型也是自然语言处理基础任务之一，大量的研究从 n 元语言模型和神经语言模型等不同角度开展了系列工作。

2.3 N-gram语言模型

  • 随着历史单词数量增多，绝大多数的历史并不会在训练数据中出现，这也意味着 P(wi∣w1w2...wi−1)P(w_i|w_1w_2...w_{i-1})P(wi​∣w1​w2​...wi−1​) 就很可能为 0，使得概率估计失去了意义。为了解决上述问题， 可以进一步假设任意单词 wiw_iwi​ 出现的概率只与过去 n−1n-1n−1 个词相关：

P(wi∣w1w2…wi−1)=P(wi∣wi−(n−1)wi−(n−2)…wi−1)可写作：P(wi∣w1i−1)=P(wi∣wi−n+1i−1)P(w_i|w_1w_2…w_{i-1}) = P(w_i|w_{i-(n-1)}w_{i-(n-2)}…w_{i-1}) \\ \text{可写作：} P(w_i|w_1^{i-1}) = P(w_i|w_{i-n+1}^{i-1}) P(wi​∣w1​w2​…wi−1​)=P(wi​∣wi−(n−1)​wi−(n−2)​…wi−1​)可写作：P(wi​∣w1i−1​)=P(wi​∣wi−n+1i−1​)

  • 满足上述条件的模型被称为 N 元语法或 N 元文法 (N-gram) 模型。其中 N-gram 表示 N 个连续单词构成的单元，也被称为 N 元语法单元。
    • 当 n=1 时，即出现在第 iii 位上的基元 wiw_iwi​ 独立于历史。一元文法也被写为 uni-gram 或 monogram；
    • 当 n=2 时，2-gram (bi-gram) 被称为1阶马尔可夫链；
    • 当 n=3 时，3-gram (tri-gram) 被称为2阶马尔可夫链，依次类推。
  • 为了保证条件概率在 i=1i=1i=1 时有意义，同时为了保证句子内所有字符串的概率和为 111， 即 ∑sp(s)=1\sum_s p(s) = 1∑s​p(s)=1，可以在句子首尾两端增加两个标志： w1w2…wmw_1w_2…w_mw1​w2​…wm​ 。
    • 不失一般性， 对于 n>2 的 n-gram， p(s) 可以分解为：p(s)=∏i=1m+1p(wi∣wi−n+1i−1)p(s) = \prod_{i=1}^{m+1}p(w_i|w_{i-n+1}^{i-1})p(s)=∏i=1m+1​p(wi​∣wi−n+1i−1​)
    • 其中，wijw_i^jwij​ 表示词序列 wi…wjw_i…w_jwi​…wj​，wi−n+1w_{i-n+1}wi−n+1​ 从 w0w_0w0​ 开始，w0w_0w0​ 为 ，wm+1w_{m+1}wm+1​ 为 。

  • 如何选择 n 的大小？

    • 当 n 较大时，提供了更多的语境信息，语境更具区别性。但是，参数个数多、计算代价大、训练语料多、参数估计不可靠；
    • 当 n 较小时，语境信息少，不具区别性；但是，参数个数少、计算代价小、训练语料无需太多，参数估计可靠；

  • 语言建模难点的解决方法

    • 设法减少历史基元的个数， 将 w1w2…wi−1w_1w_2…w_{i-1}w1​w2​…wi−1​ 映射到等价类 S(w1w2…wi−1)S(w_1w_2…w_{i-1})S(w1​w2​…wi−1​)，使等价类的数目远远小于原来不同历史基元的数目。则有：p(wi∣w1w2⋯wi−1)=p(wi∣S(w1⋯wi−1))p(w_i|w_1 w_2 \cdots w_{i-1}) = p(w_i|S(w_1 \cdots w_{i-1}))p(wi​∣w1​w2​⋯wi−1​)=p(wi​∣S(w1​⋯wi−1​))；
    • 如何划分等价类？其中一种方法是：将两个历史映射到同一个等价类， 当且仅当这两个历史中的最近 n-1 个基元相同。

  • 平滑方法：尽管 n 元语言模型能缓解句子概率为 0 的问题，但语言是由人和时代创造的，具备无穷的可能性，再庞大的训练语料也无法覆盖所有的 n-gram， 而训练语料中的零频率并不代表零概率。因此，需要使用平滑技术 (Smoothing) 来解决这一问题；

    • 平滑是指为了产生更合理的概率，对最大似然估计进行调整的一类方法，也称为数据平滑 (Data Smoothing)。平滑处理的基本思想是提高低概率，降低高概率，使整体的概率分布趋于均匀；
    • 加法平滑 (Additive Smoothing) 是实际运用中最常用的平滑技术之一，其思想是假设事件出现的次数比实际出现的次数多 δδδ 次。当 δ=1δ = 1δ=1 时， 方法又称为加一平滑；
    • 此外还有：古德-图灵估计法、Katz 平滑、Jelinek Mercer 平滑算法、Witten-Bell 平滑算法、Kneser-Ney 平滑算法等。很多算法的核心思想可以概括为：如果 n-gram 存在则使用其本身计数，如果不存在再退后到低阶分布。

  • n 元语言模型存在的问题

    • 不能在相似的单词之间分享权重
    • 不能以插入单词的上下文为条件
    • 不能处理长期依赖问题

2.4 神经网络语言模型

  • 随着深度神经网络的发展，利用神经网络的语言模型展现出了比 n 元语言模型更强学习能力。神经网络先进的结构使其能有效的建模长距离上下文依赖，以词向量 (Word Embedding) 为代表的分布式表示的语言模型深刻地影响了自然语言处理领域的其他模型与应用的变革。n元语言模型几乎已经被神经网络的语言模型所替代。

3 文本表示

  • 如何表示一个单词的意义？

    • WordNet：使用包含同义词集和上位词 (‘is a’关系) 的列表

      • 存在的问题：缺少细微差别、缺少单词的新含义、主观的、需要人工劳动来创建和调整、无法用于准确计算词语相似度

    • Word 离散表示：将单词视为离散符号，如独热编码 (one-hot)，其中向量维度 = 词汇表中单词的数量

      • 存在的问题：所有向量都是正交的，没有相似性概念

    • 通过上下文来表示单词：A word’s meaning is given by the words that frequently appear close-by

      • 现代统计自然语言处理中最成功的思想之一
      • 当一个单词 www 出现在文本中时，它的上下文是指出现在附近的一组单词 (在一个固定大小的窗口内)

3.1 词向量

  • 通过上下文 (Context) 来表示单词
    • 语言模型的思想是基于历史的词 (上文)，去预测下一个词 (下文)。类似的，语言模型的变种也可以是：基于下文，预测上文。Context words 是某个单词的上下文，有较为充足的信息表示该单词。
  • 词向量 (word vectors / word embeddings)
    • 我们将为每个单词构建一个低维、稠密的向量，使出现在相似上下文中的单词，向量也相似
    • 通过衡量向量点积来度量相似性

3.2 词袋模型

  • 向量空间模型 (Vector Space Model, VSM) 由 G. Salton 等人于1960s末期在信息检索领域提出，核心是将文本视为特征项的集合：
    • 特征项是 VSM 中最小的语言单元，可以是字、词、短语等；若词语作为特征项，那么特征项的集合可视为一个词表。词表可从语料中统计获得，可看做一个词袋，向量空间模型被称为词袋模型 (bag of words, BOW)；
    • 文本表示为特征项的集合 (t1,t2,t3,⋯ ,tn)(t_1,t_2, t_3,\cdots,t_n)(t1​,t2​,t3​,⋯,tn​)；
    • 特征项权重：每个特征项在文本中的重要性不尽相同，用 www 表示特征项 ttt 的权重，相应地，文本可以表示为 (t1:w1,t2:w2,⋯ ,tn:wn)(t_1:w_1, t_2:w_2, \cdots,t_n:w_n)(t1​:w1​,t2​:w2​,⋯,tn​:wn​) 或 (w1,w2,⋯ ,wn)(w_1, w_2, \cdots,w_n)(w1​,w2​,⋯,wn​)；
    • 权重计算：布尔变量 (是否出现过)、词频、逆文档频率、TF-IDF等
  • 出现的问题
    • 文本长度对文本表示有影响；
    • 稀疏性：对于大词典，尤其是包括了生僻字的词典， 文档稀疏性不可避免；
    • 多义词：BOW 模型只统计单词出现的次数，而忽略了一个词可能存在多种含义；
    • 同义词：多个单词可以表示同一个意思；

5.2 梯度消失

  • 梯度消失：
    • 梯度信号随着进一步反向传播会变得越来越小；
    • 来自远处的梯度信号会丢失，因为它比来自附近的态度信号小得多；
    • 因此，模型权重基本上仅针对近期效应而不是长期效应进行更新。

5.3 梯度爆炸

  • 如果梯度变得太大，那么 SGDSGDSGD 更新步长变得太大，这可能会导致糟糕的更新。在最坏的情况下，可能出现 Inf 或 NaN；
  • 应对方法：梯度裁剪。如果梯度范数大于某个阈值，则对其进行缩放。

6 LSTM与Seq2Seq

  • 长短期记忆神经网络 (Long Short-Term Memory, LSTM) 历史：
    • Hochreiter 和 Schmidhuber 在 1997 年提出的一种 RNN，用于解决梯度消失问题；
    • 但现代 LSTM 的真正关键部分来自于 2000 年的 Gers 和 Schmidhuber 等；

    • 后来 Graves 成为 Hinton 的博士后，Hinton 在 2013 年将 LSTM 引入 Google，LSTM 这才广为人知。

6.1 LSTM结构

  • LSTM 的关键在于细胞状态，即图表顶部横穿过去的水平线；
    • 细胞状态像传送带贯穿整个链条，只有一些微小的线性交互作用。信息很容易沿着它不变地流动。
  • LSTM 通过被称为门 (gate) 的结构调节，来移除或添加信息到细胞状态中；
    • sigmoid 层输出的数字介于 0 到 1 之间，描述了应该让每个组件通过多少：0 的值意味着"什么也不让通过"，而 1 的值意味着"让一切通过"；
    • 遗忘门由一个 sigmoid 神经网络层和一个逐点乘法操作组成。它查看 ht−1h_{t-1}ht−1​ 和 xtx_txt​，并且对细胞状态 Ct−1C_{t-1}Ct−1​ 中的每个数字输出一个介于 0 和 1 之间的数字，决定哪些信息被保留，哪些被遗忘；此外，遗忘门还缩放了前一个单元的状态，防止了单元状态的无限制增长；
    • 输入门的 sigmoid 层决定我们将更新哪些值。此外，一个 tanh 层创建一个新的候选值向量 C~t\tilde{C}_tC~t​， 这些值可以被添加到状态中。 现在对状态的更新：Ct=ft∗Ct−1+it∗C~tC_t = f_t*C_{t-1} + i_t * \tilde{C}_tCt​=ft​∗Ct−1​+it​∗C~t​；
    • 输出门首先运行一个 sigmoid 层，决定输出细胞状态的哪些部分。然后，将细胞状态通过 tanh 将值推至 -1 和 1 之间，并将其乘以 sigmoid 门的输出，这样我们只输出我们决定输出的部分 hth_tht​。

    • ot,ft,ito_t,f_t,i_tot​,ft​,it​ = 时间 ttt 的 LSTM 输出、遗忘和输入门
    • xt,ytx_t,y_txt​,yt​ = 时间 ttt 的输入和输出
  • LSTM 架构使得 RNN 更容易保存多个时间步长的信息
    • 例如，如果某个单元维度的遗忘门设置为 1，而输入门设置为 0，则该单元的信息将无限期保留；
    • 相比之下，普通 RNN 更难学习在隐藏状态中保留信息的循环权重矩阵 WhW_hWh​；
    • 在实践中，会保留大约 100 个 steps，而不是经典 RNN 的大约 7 个；
    • 但是，还有其他方法可以在模型中创建更直接和线性的直通连接以实现长距离依赖 (如，ResNet 中的残差网络、Attention 机制等)。
  • 梯度消失/爆炸不只是 RNN 的问题
    • 尤其对于较深的神经网络，它们的较低层的学习速度非常慢，很难训练；
    • 其他解决方案：
      • ResNet：直接连接，从而允许梯度流动；
      • DenseNet：密集连接，将每一层连接到所有未来层；
      • HighwayNet：高速公路连接，使用动态门控制恒等连接与转换层。

6.4 RNN/LSTM应用

  • 词性标注
  • 情感分类
  • 生成其他信息，如语音识别、机器翻译、摘要等

6.5 双向RNN

  • 双向 RNN 仅适用于有权访问整个输入的情况
    • 不适用于语言建模，因为在 LM 中只剩下上下文可用；
    • 如果确实有完整的输入序列，则双向性非常强大 (应该默认使用)；
    • BERT (Bidirectional Encoder Representations from Transformers) 就是一个基于双向性的强大预训练上下文表示系统。
